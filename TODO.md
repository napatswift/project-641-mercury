    - [ ]  หากโปรโมชันที่ระบุไม่สอดคล้องกับการเงื่อนไขในการใช้โปรโมชันจะต้อง
      - [ ]  มีข้อความแจ้งเตือนที่เหมาะสม 
      - [ ]  มีข้อความแจ้งเตือนที่เข้าใจได้ว่าทำไมจึงใช้โปรโมชันที่กรอกโค้ดนี้ไม่ได้
## ความสวยงามและประสบการณ์ของผู้ใช้งาน
  - [ ] (5 คะแนน) ใช้โทนสีและองค์ประกอบต่าง ๆ ของ GUI ที่ดูได้ชัดเจน น่าใช้ ไม่แสบตา ไม่ลวงตา โดยลองนึกถึงว่าหากเราทำโปรแกรมนี้ให้คนทั่วไปใช้จริง ควรแสดงองค์ประกอบเหล่านี้อย่างไร
  - (caution) โปรดตรวจสอบความถูกต้องของข้อความที่ปรากฏในส่วนต่าง ๆ ของโปรแกรม หรือเอกสารประกอบ หากพบว่ามีคำผิด หักคำที่ผิดจุดละ 5 คะแนน
  - [ ] **(10 คะแนน) Graphic User Interface (GUI)**
    - ควรมีรูปแบบที่เข้าใจง่าย มีการแสดงข้อมูลที่ชัดเจน 
    - [ ]  มี Label กำกับข้อมูลที่ชัดเจน เพื่อให้เข้าใจว่าค่าที่แสดงนั้นคือค่าอะไร 
    - มีการแสดงองค์ประกอบต่าง ๆ ในขนาดที่เหมาะสม ไม่เล็กหรือไม่ใหญ่จนเกินไป 
    - [ ]  ขนาดหน้าจอของโปรแกรมต้องมีความสูงเกิน 1024 pixel
  - [ ]  (extra 5 คะแนน) GUI มี effect ที่น่าสนใจ
    - เมื่อมี action ต่าง ๆ เช่น มีการ highlight button เมื่อคลิกปุ่ม 
    - มี effect การ transition เมื่อเปลี่ยน scene เป็นต้น
  - [ ] (extra 10 คะแนน) Theme
    - [ ] ผู้ใช้โปรแกรมสามารถปรับเปลี่ยน Theme ของ Application ได้
    - เช่น เปลี่ยนโทนสี เปลี่ยนขนาดอักษร 
    - หรือเปลี่ยนฟอนต์ของตัวอักษรได้ 
    - [ ] โดยจะต้องเปลี่ยนให้สอดคล้องกันทุก Scene
## บันทึก
โปรแกรมต้องสามารถบันทึกสถานะของข้อมูลต่าง ๆ ในรูปแบบของไฟล์ csv (comma-separated values) และสามารถโหลดไฟล์ที่บันทึกไว้นั้นมาแสดงผลได้อย่างถูกต้อง
  - [x] (caution) ต้องมีการบันทึกสถานะและข้อมูลที่จำเป็นในรูปแบบไฟล์ csv
  - [x] **(5 คะแนน) เมื่อเปิดโปรแกรมใหม่ หรือเมื่อโปรแกรมโหลดไฟล์ csv จะต้องแสดงข้อมูลที่ได้บันทึกไว้อย่างถูกต้อง**

## ข้อมูลในที่ใช้การตรวจสอบ
  ต้องมีข้อมูลเริ่มต้นในการทดสอบระบบ เสมือนว่าระบบถูกใช้งานมาประมาณ 1 เดือน เป็นอย่างน้อย
  - [ ] ระบบถูกใช้งานมาประมาณ 1 เดือน
  - [ ] มีข้อมูลผู้ดูแลระบบ ข้อมูลบัญชีผู้ใช้งานอย่างน้อย 5 บัญชี 
  - [ ] มีร้านค้าอย่างน้อย 3 ร้านค้า 
  - [ ] มีสินค้าอย่างน้อย 10 รายการ
 
  หากมีข้อมูลเริ่มต้นระบบน้อยกว่าที่กำหนด จะไม่ตรวจให้
